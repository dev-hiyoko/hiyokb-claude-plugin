# handoff — セッション引き継ぎサマリ

別セッションがドシエを読めば続きから再開できる状態にする（このプラグインの存在理由の半分）。

## 対象の特定
- `$ARGUMENTS` でタスクが指定されればそれ。無ければ `~/.task/index.md` から **status が in_progress / blocked** のタスクを対象にする（複数なら各々）。

## 各タスクで行うこと
1. ドシエ（`~/.task/projects/<名>/tasks/<id>.md`）を Read。
2. 以下を簡潔にまとめる:
   - **現在地**: 手順のどのステップまで完了したか（チェックリスト）。
   - **直近の決定**: 決定事項から要点。
   - **次アクション**: 次にやること1つ。
   - **ブロッカー**: blocked なら「何待ち＋解除条件」。
3. ドシエの **「## 進捗ログ」末尾に日付つきで追記**（追記専用）:
   ```
   - <YYYY-MM-DD> handoff: <現在地>。次は <次アクション>。<ブロッカーがあれば明記>
   ```
   日付は `date +%F` で取得。
4. frontmatter の `updated` を今日に更新。

## 出力
- 画面には**引き継ぎブロック**（タスクごとに 現在地 / 決定 / 次アクション / ブロッカー）を簡潔に提示。
- これでドシエが単体で文脈復元できる状態か確認する。

## 原則
- 会話の記憶でなく**ドシエに書く**ことで引き継ぐ。
- 破壊的なことはしない（記録のみ）。`sensitive` タスクの内容は外部送信しない。
