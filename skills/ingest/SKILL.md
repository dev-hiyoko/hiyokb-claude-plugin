# ingest — 情報の取り込み・要約・振り分け

ユーザー提供の素材を取り込み、知識として整理する。**原本は不変**（読むだけ・書き換えない）。

## 1. 原本を inbox に取り込む（不変保存・冪等）
入力の形に応じて取り込みスクリプトを使う。出力された inbox 相対パスを控える（`EXISTS\t...` なら既取り込み＝重複させない）。

- **ファイル/画像のパスが渡された場合**:
  ```bash
  python3 "${CLAUDE_PLUGIN_ROOT}/scripts/ingest_intake.py" "<ファイルパス>"
  ```
- **テキストが貼り付けられた場合**（タイトルを付ける）:
  ```bash
  python3 "${CLAUDE_PLUGIN_ROOT}/scripts/ingest_intake.py" --text --title "<短いタイトル>" <<'EOF'
  <貼り付けられた本文>
  EOF
  ```
これで `~/.task/inbox/<日付>_<名前>` に保存され、`~/.task/log.md` に取り込みが記録される。

## 2. 内容を読む
- 取り込んだ inbox のファイルを Read する。
- **画像**はマルチモーダルで読み取り、写っている要点（UI/エラー/図表/ホワイトボード等）を抽出。
- **文字起こし/メモ**は通読して要点を掴む。

## 3. 要約・抽出
- 簡潔な要約、**決定事項**、**アクションアイテム(TODO)**、登場する人物/用語を整理。
- **出自(provenance)を必ず保持**: 出所（元ファイル名/日付/発言者）を残す。

## 4. 振り分け（プロジェクト & 行き先）
- どのプロジェクトの話か判断。**不明ならユーザーに確認**（勝手に決めない）。
- 行き先:
  - 既存タスクに関係 → 該当ドシエの「調査メモ」「決定事項」「次アクション」に追記。
  - 新しいタスクが必要 → ドシエ新規作成を提案（`/hiyokb:task-focus` に繋ぐ）。
  - 再利用可能な知識 → `~/.task/kb/`（または `projects/<名>/kb/`）への追記を提案。

## 5. 昇格ゲート（Evidence Gate）
取り込んだ内容を無条件に知識化しない。

- **自動で昇格してよい**: 出所が明確（元ファイル/発言者/日付が特定でき）、既存の記述と矛盾しない確定情報。
- **昇格せず inbox に留める / draft 扱い**: 出所が曖昧、未確定、噂・憶測、要約途中。`config.yaml` の `promotion.auto` が `always_confirm` なら**必ず人に確認**してから昇格。
- 昇格したら、要約ページ/追記に出所を明記する。

## 6. 機密の扱い
- 文字起こし等は社外秘を含みうる。機密と判断したら、作成するドシエ/メモに `sensitive: true` を立て、**外部送信を伴う処理（Web調査・外部連携）を避ける**。
- 原本（inbox）は `~/.task/.gitignore` で同期対象外。

## 原則
- 原本は決して書き換えない（加工結果は別ファイルに書く）。
- 同じ素材を二重取り込みしない（スクリプトの冪等チェックに従う。`EXISTS` なら既存を使う）。
