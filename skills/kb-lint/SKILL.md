# kb-lint — ナレッジベースの健全性チェック

KB の価値はリンク（グラフ）に宿るため、**リンク健全性を最重視**して点検する。

## 検出のみ（既定）
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/kb_lint.py"
```
出力される指摘:
- **リンク切れ**: `[[X]]` の参照先ページが無い（※人手で修正）
- **孤立ノート**: どこからもリンクされていないページ → 結び直しを提案
- **片方向リンク**: A→B はあるが B→A が無い
- **出自なし**: `source` が無いページ
- **relates_to の参照先なし**

## 自動修復（--fix・機械的に安全なものだけ）
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/kb_lint.py" --fix
```
- KB 索引（index.md）を再生成
- 片方向リンクを相互リンク化（対象ページの「## 関連」に `[[元ページ]]` を冪等に追記）
- **リンク切れの張り直しや矛盾の解消は自動でやらない**（意図を機械判断できないため）。報告に従い人が判断。

## 仕上げ
- 検出結果をユーザーに分かりやすく要約し、**特に孤立・矛盾**は「どう結ぶ/どちらを正とするか」を相談する。
- 矛盾（contradicts）や陳腐化した古い主張は、`/hiyokb:kb` で該当ページを改訂し `supersedes` で関係づける。
