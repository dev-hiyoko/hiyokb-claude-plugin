# bugfix（バグ修正）

**適用**: 既存挙動が仕様/期待と異なる。「治す」が主目的。

## 手順
- [ ] 0. ドシエ確認（前回の進捗ログ・決定事項を読み再開 / 初回は概要・AC を把握）
- [ ] 1. 再現手順を確定（DoD: 手元で再現できる / 調査メモに環境・条件を記録）
- [ ] 2. 原因の切り分け（DoD: 原因箇所を1つに特定 / 決定事項に根拠つきで記録）
- [ ] 3. 修正方針の決定（DoD: 影響範囲と代替案を比較し方針を決定事項に）
- [ ] 4. 修正＋回帰テスト追加（DoD: バグを捉えるテストが赤→緑）
- [ ] 5. セルフレビュー・影響範囲確認（DoD: 周辺・類似箇所も確認）
- [ ] 6. PR/MR等の提出（DoD: レビュアー指定 / 関連 Issue リンク。提出先が無いタスクは完了報告）
- [ ] 7. 振り返り（効いた手順・ハマりどころを抽出 → 汎用なら KB 昇格）

## Tips / 落とし穴
- 「再現できない」段階で修正に進まない（最大の時間浪費）。
- 同じ原因の類似バグが他にないか（横展開）を 5 で必ず確認。
- 根本原因と対症療法を区別し、決定事項にどちらかを明記。
