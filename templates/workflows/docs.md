# docs（ドキュメント作成・更新）

**適用**: 仕様書・手順書・README・設計メモ等の作成/更新。

## 手順
- [ ] 0. ドシエ確認（前回の進捗ログ・決定事項を読み再開 / 初回は目的を把握）
- [ ] 1. 読者と目的の確定（DoD: 誰が何のために読むかを冒頭に）
- [ ] 2. アウトライン（DoD: 見出し構成を先に合意）
- [ ] 3. 本文作成（DoD: 結論先行・具体例つきで記述）
- [ ] 4. 正確性の検証（DoD: 実際の挙動/コードと突き合わせ）
- [ ] 5. リンク・配置（DoD: 関連ドキュメント/KBと相互リンク）
- [ ] 6. 振り返り（再利用できる雛形・知見は KB へ）

## Tips / 落とし穴
- 「何を書くか」より先に「誰向けか」。粒度がぶれる原因。
- 陳腐化対策として、出典・更新日・対象バージョンを残す。
