# feature（新機能・機能追加）

**適用**: 新しい価値/挙動を足す。

## 手順
- [ ] 0. ドシエ確認（前回の進捗ログ・決定事項を読み再開 / 初回は概要・AC を把握）
- [ ] 1. 要求と受け入れ条件の確定（DoD: AC をドシエの受け入れ条件節に箇条書き）
- [ ] 2. 既存実装・関連箇所の調査（DoD: 触る範囲と再利用できる部品を調査メモに）
- [ ] 3. 設計（DoD: データ/IF/段取りを決定事項に。必要なら代替案比較）
- [ ] 4. 最小実装（垂直スライス）（DoD: 一連の流れが1本通る）
- [ ] 5. テスト（正常+境界+異常）（DoD: AC を満たすテストが緑）
- [ ] 6. ドキュメント/設定/移行の手当て（DoD: 使う側が分かる状態）
- [ ] 7. PR/MR等の提出（DoD: レビュアー指定。提出先が無いタスクは完了報告）
- [ ] 8. 振り返り（効いた手順・ハマりどころを抽出 → 汎用なら KB 昇格）

## Tips / 落とし穴
- 1 の AC が曖昧なまま実装しない。曖昧なら着手前にユーザ確認。
- 4 で大きく作り込まず、まず end-to-end を細く通す。
- スコープのにじみ出し（やりすぎ）に注意。AC 外は別タスク化。
