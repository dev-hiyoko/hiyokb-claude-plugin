# research（調査・技術検証・PoC）

**適用**: 答え/方針が未確定で、まず「調べて決める」のが目的。

## 手順
- [ ] 0. ドシエ確認（前回の進捗ログ・決定事項を読み再開 / 初回は問いの背景を把握）
- [ ] 1. 問い（answerable question）と完了条件を定義（DoD: 「何が分かれば終わりか」をドシエ冒頭に）
- [ ] 2. 情報源の収集（DoD: コード/ドキュメント/過去ドシエ/KB/必要ならWebを調査メモに、出典つき）
- [ ] 3. 検証/比較（DoD: 仮説を実験 or 比較表で検証）
- [ ] 4. 結論と根拠（DoD: 決定事項に結論＋根拠＋却下案）
- [ ] 5. 次タスク化（DoD: 結論を実装/対応タスクに落とす）
- [ ] 6. 振り返り（再利用可能な知見を KB へ。出典明確なら昇格）

## Tips / 落とし穴
- 1 の問いを決めずに調べ始めると無限に広がる。完了条件を先に固定。
- 出典を必ず残す（昇格ゲートの前提）。
- 調査メモと確定知識を混ぜない（生 → inbox/draft、確定 → KB）。
